# ПОРУЧЕНИЕ НА ОБРАБОТКУ ПЕРСОНАЛЬНЫХ ДАННЫХ (DPA) ONLAIS

Редакция: 1.0  
Дата вступления в силу: 19.04.2026  
Адрес публикации: https://onlais.ru/dpa

Настоящее Поручение является неотъемлемой частью Пользовательского соглашения ONLAIS.

## 1. Стороны и правовой статус

1.1. Платформа ONLAIS (ОНЛАЙС) предоставляется ИП Колеватых Виталием Александровичем (ИНН 245510825785, ОГРНИП 325784700306879) (далее — «Обработчик»).

1.2. Пользователь платформы, использующий ONLAIS для обработки данных своих клиентов, является оператором персональных данных (далее — «Оператор данных пользователя»).

1.3. Настоящее Поручение определяет условия, при которых Обработчик обрабатывает персональные данные по поручению Оператора данных пользователя.

## 2. Предмет поручения

2.1. Оператор данных пользователя поручает, а Обработчик принимает на себя обязанность обрабатывать персональные данные в объеме, необходимом для предоставления функционала ONLAIS.

2.2. Обработка осуществляется исключительно в рамках исполнения договорных обязательств по предоставлению платформы и связанных сервисов.

## 3. Категории субъектов и персональных данных

3.1. Категории субъектов:
- клиенты Оператора данных пользователя;
- потенциальные клиенты;
- сотрудники/представители Оператора данных пользователя (при наличии соответствующих данных).

3.2. Примерные категории данных:
- ФИО;
- номера телефонов;
- email;
- информация о записях, услугах, заказах, истории посещений;
- служебные комментарии и заметки;
- иные данные, введенные Оператором данных пользователя.

3.3. Обработчик не предназначен для обработки специальных категорий персональных данных, биометрических данных и иных чувствительных категорий, если иное не предусмотрено законом и отдельными соглашениями.

## 4. Цели обработки

Обработка по настоящему Поручению осуществляется для целей:
- организации онлайн-записи и расписания;
- ведения CRM и клиентской базы;
- коммуникации с клиентами;
- поддержки работы сайта/виджетов/интеграций;
- обеспечения информационной безопасности;
- формирования статистики и аналитики в интересах Оператора данных пользователя.

## 5. Перечень действий по обработке

Обработчик вправе выполнять следующие действия с персональными данными:
- сбор, запись, систематизация, накопление, хранение;
- уточнение (обновление, изменение);
- извлечение, использование;
- передача (предоставление доступа) в предусмотренных случаях;
- обезличивание;
- блокирование;
- удаление и уничтожение.

## 6. Инструкции Оператора данных пользователя

6.1. Инструкциями Оператора данных пользователя являются:
- условия настоящего Поручения;
- условия Пользовательского соглашения ONLAIS;
- настройки и команды, передаваемые через интерфейс платформы;
- документированные запросы Оператора данных пользователя в поддержку.

6.2. Обработчик не использует персональные данные для целей, не предусмотренных документированными инструкциями и договорными обязательствами.

## 7. Обязанности Обработчика

Обработчик обязуется:
- обеспечивать конфиденциальность персональных данных;
- обрабатывать данные только в пределах инструкций и применимого законодательства;
- применять организационные и технические меры защиты;
- ограничивать доступ к данным только уполномоченным лицам;
- обеспечивать журналирование и контроль доступа;
- уведомлять Оператора данных пользователя об инцидентах безопасности без неоправданной задержки;
- по запросу предоставлять информацию, подтверждающую выполнение обязанностей по настоящему Поручению.

## 8. Обязанности Оператора данных пользователя

Оператор данных пользователя обязуется:
- иметь законные основания для обработки персональных данных;
- получать необходимые согласия субъектов (если это требуется законом);
- обеспечивать соблюдение требований законодательства РФ при обработке персональных данных;
- самостоятельно определять цели и состав обрабатываемых данных;
- не передавать в ONLAIS данные, обработка которых запрещена законом или не обеспечена необходимыми основаниями.

## 9. Привлечение субобработчиков

9.1. Оператор данных пользователя предоставляет Обработчику общее разрешение на привлечение субобработчиков, необходимых для функционирования платформы (инфраструктура, хостинг, коммуникации, аналитика, безопасность и др.).

9.2. Обработчик обеспечивает, чтобы на субобработчиков возлагались обязательства по защите данных не ниже, чем предусмотренные настоящим Поручением.

## 10. Передача и трансграничная обработка

10.1. Передача персональных данных третьим лицам допускается только в объеме, необходимом для выполнения договорных обязательств, при наличии правовых оснований.

10.2. Трансграничная передача допускается только при соблюдении требований законодательства РФ и наличии законных оснований.

## 11. Безопасность обработки

11.1. Обработчик применяет комплекс мер безопасности, включая:
- защищенные каналы передачи данных;
- антивирусную и сетевую защиту;
- резервное копирование и восстановление;
- контроль доступа и разграничение прав;
- аудит событий и мониторинг инцидентов.

11.2. Конкретный состав мер может актуализироваться Обработчиком с учетом развития технологий и угроз безопасности.

## 12. Инциденты безопасности

12.1. При выявлении инцидента, повлекшего риск неправомерного доступа, раскрытия, изменения, блокирования, удаления или уничтожения персональных данных, Обработчик уведомляет Оператора данных пользователя без неоправданной задержки.

12.2. Уведомление содержит, насколько это возможно:
- описание характера инцидента;
- потенциальные последствия;
- уже принятые и/или планируемые меры реагирования.

## 13. Помощь в реализации прав субъектов

13.1. С учетом технических возможностей платформы Обработчик оказывает разумное содействие Оператору данных пользователя в обработке запросов субъектов персональных данных.

13.2. Основная обязанность по взаимодействию с субъектами персональных данных лежит на Операторе данных пользователя.

## 14. Срок обработки, удаление и возврат данных

14.1. Обработка данных осуществляется в течение срока действия договорных отношений и/или до получения инструкции на удаление, если иное не установлено законом.

14.2. По запросу Оператора данных пользователя Обработчик удаляет или возвращает данные в разумный срок, если отсутствует обязанность хранения данных по закону.

14.3. При прекращении договорных отношений данные подлежат удалению, обезличиванию или иному законному режиму хранения.

## 15. Ответственность сторон

15.1. Оператор данных пользователя несет ответственность как оператор персональных данных за законность сбора и обработки данных субъектов.

15.2. Обработчик несет ответственность за соблюдение своих обязательств как обработчика в пределах, предусмотренных договором и законом.

15.3. Ни одно положение настоящего Поручения не освобождает стороны от ответственности, которую невозможно ограничить в силу императивных норм закона.

16.2. Обработчик вправе изменять Поручение в одностороннем порядке, размещая новую редакцию на сайте.

16.3. Новая редакция применяется с момента публикации, если иной срок не указан в самой редакции.
