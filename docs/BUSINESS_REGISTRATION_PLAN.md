# План: регистрация бизнеса в ONLAIS

Статус: `v3`  
Дата обновления: `2026-04-19`

## 1. Цель

Сделать корректную самостоятельную регистрацию нового бизнеса в CRM ONLAIS:
- вход по `email + пароль` (без ручного `accountSlug`),
- выбор организационно-правовой формы и сферы деятельности,
- обязательные юридические согласия при регистрации,
- автоматическое создание аккаунта и сайта-черновика,
- жесткая уникальность owner-контактов (email/телефон).

## 2. Фиксированные бизнес-правила

- Один владелец бизнеса = один уникальный `email` и один уникальный `телефон`.
- Нельзя зарегистрировать новый аккаунт, если этот `email` уже используется в другом аккаунте владельца.
- Нельзя зарегистрировать новый аккаунт, если этот `телефон` уже используется в другом аккаунте владельца.
- Проверка уникальности делается до финального создания аккаунта и дублируется ограничениями БД.
- Публичная ссылка (`slug`) создается автоматически; в форме регистрации пользователь ее не вводит.
- Авторизация в CRM: только `email + пароль`; выбор аккаунта появляется только если у пользователя несколько аккаунтов.

## 3. Целевой UX регистрации

1. Шаг 1: `Email + пароль`.
2. Шаг 2: подтверждение email (код/ссылка).
3. Шаг 3: данные бизнеса:
   - название,
   - форма: `Частный специалист / Самозанятый / ИП / ООО`,
   - сфера деятельности,
   - телефон,
   - часовой пояс.
4. Шаг 4: обязательные согласия (чекбоксы).
5. Шаг 5: создание аккаунта, автологин, создание стартового сайта.

## 5. Backend API

### 5.1 Новые endpoints

- [ ] `POST /api/v1/crm/auth/register/start`
  - вход: `email`, `password`
  - результат: отправка кода/ссылки подтверждения

- [ ] `POST /api/v1/crm/auth/register/verify-email`
  - вход: токен/код
  - результат: email подтвержден

- [ ] `POST /api/v1/crm/auth/register/complete`
  - вход: бизнес-данные + согласия
  - транзакция:
    - [ ] создать `User`/identity,
    - [ ] создать `Account` и `slug`,
    - [ ] назначить роль `OWNER`,
    - [ ] создать сайт-черновик,
    - [ ] зафиксировать согласия,
    - [ ] создать сессию.

- [ ] `POST /api/v1/crm/auth/select-account` (для мультиаккаунта).

### 5.2 Обновление логина

- [ ] `POST /api/v1/crm/auth/login` без `accountSlug`.
- [ ] Если аккаунтов > 1, вернуть список аккаунтов для выбора.
