2) Продукт (единая экосистема) — ЭТАЛОН

2.0. Принцип продукта
У нас 4 зоны в одной экосистеме, но ядро одно:
- Marketplace — поиск/витрина/профили/вход в запись
- Client — личный кабинет клиента
- CRM Business — операционка бизнеса
- Platform Admin — управление платформой

Ключевое правило: все 4 зоны используют один Booking Engine, одну модель настроек, один каталог событий и один API контракт.

2.1. Marketplace (Web + Mobile)
2.1.1. Цель
- просмотр карточек/профилей
- выбор услуги/мастера/даты
- создание записи
- сохранение избранного
- переход в Client

2.1.2. Основные разделы Marketplace
- Лента (каталог)
- Карта
- Поиск
- Профили (аккаунт/локация/специалист/услуга)

2.1.3. Фильтры (единый стандарт)
- город/район
- направления (категории)
- услуги
- цена “от”
- рейтинг
- свободно сегодня/завтра
- онлайн-оплата
- акции

2.1.4. Онлайн-запись
- Marketplace только входная точка
- единая логика слотов (Booking Engine)

2.1.5. SEO (Web Marketplace)
- sitemap/robots
- OG meta
- schema.org
- canonical URLs
- slug rules и редиректы

2.2. Client App (Web + Mobile)
2.2.1. Цель
- записи (создать/перенести/отменить/повторить)
- оплаты/чеки/возвраты
- избранное
- лояльность
- уведомления и настройки каналов

2.2.2. Разделы Client
- Auth
- Профиль
- Мои записи
- Оплаты
- Избранное
- Лояльность
- Уведомления
- Настройки уведомлений

2.3. CRM Business (Web + Mobile)
2.3.1. Цель
- календарь/записи
- расписание персонала
- услуги/цены/уровни
- клиенты
- финансы (по правам)
- аналитика
- настройки

2.3.2. Обязательные разделы CRM
- Dashboard
- Календарь записей
- Расписание
- Услуги
- Специалисты
- Клиенты
- Оплаты/Финансы
- Промо/скидки
- Лояльность
- Аналитика
- Настройки

2.4. Platform Admin (Web + Mobile)
2.4.1. Цель
- аккаунты/лимиты/модули
- тарифы/подписки/платежи
- модерация
- мониторинг
- аудит
- платформенные настройки

2.4.2. Обязательные разделы Platform Admin
- Accounts
- Plans / Billing
- Moderation
- Monitoring
- Audit
- Platform Settings

2.5. Единые продуктовые стандарты
- состояния UI: loading/empty/error/success
- deep links
- совместимость модулей (тарифы/модули/настройки/права)
