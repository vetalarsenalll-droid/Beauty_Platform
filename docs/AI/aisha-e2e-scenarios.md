# Aisha E2E Scenarios

## Purpose
Покрыть ключевые ветки: chat-only, booking-flow, client-actions, rollback, confirmations, post-completion.

## Chat-only
| ID | Input | Expected |
|---|---|---|
| C-01 | `привет` | Живое приветствие, без принудительного выбора локации |
| C-02 | `как дела?` | Smalltalk-ответ, без старта записи |
| C-03 | `кто ты?` | Представляется как Аиша |
| C-04 | `что умеешь?` | Короткий список возможностей (запись/перенос/отмена/контакты) |
| C-05 | `какое число сегодня?` | Отвечает текущей датой в формате `YYYY-MM-DD` |
| C-06 | `дай номер` | Возвращает телефон студии из профиля (или честно: не найден) |
| C-07 | `какие мастера?` | Список мастеров аккаунта |
| C-08 | `что по услугам?` | Список услуг с ценой и длительностью |

## Booking-flow: flexible entry
| ID | Input sequence | Expected |
|---|---|---|
| B-01 | `на завтра есть окна?` | Показывает окна по локациям, не выдумывает |
| B-02 | `в center на 17:00` | Фиксирует локацию+время, просит услугу |
| B-03 | `гель лак` | Проверяет слот и предлагает мастера/режим завершения |
| B-04 | `сам` | Возвращает action `open_booking` + статус `READY_SELF` |
| B-05 | `оформи через ассистента` -> `согласен...` -> `да` | Создает запись, статус `COMPLETED` |
| B-06 | `стрижка` при наличии men/women | Уточняет услугу (ambiguous disambiguation) |
| B-07 | `на вечер` | Фильтрация слотов по времени суток |
| B-08 | `не то, хочу другое время` | Rollback времени/мастера, повторный подбор |
| B-09 | `не на маникюр, а педикюр` | Rollback услуги+мастера+времени, корректный пересчет |
| B-10 | `спасибо` после `COMPLETED` | Не перезапускает запись |
| B-11 | `новая запись` после `COMPLETED` | Сбрасывает draft и стартует новый flow |

## Booking-flow: guards
| ID | Input | Expected |
|---|---|---|
| G-01 | Повтор `да` после успешного create | Не создает дубль |
| G-02 | Критичное действие без подтверждения | Запрашивает подтверждение |
| G-03 | Слот уже занят в момент create | Сообщает `слот занят`, предлагает другое время |
| G-04 | Невалидная дата/время | Сообщает об ошибке без падения |

## Client-actions (authorized only)
| ID | Input | Expected |
|---|---|---|
| A-01 | `покажи мои записи` | Возвращает последние N записей клиента |
| A-02 | `моя статистика` | Возвращает визиты/отмены/средний чек/топ-услугу |
| A-03 | `отменить #123` | Подтвержденная отмена доступной записи |
| A-04 | `перенести #123 на 2026-03-02 18:00` | Пытается перенос, дает результат |
| A-05 | `повтори прошлую запись` | Берет последнюю запись и просит дату/время |
| A-06 | `обнови телефон +79991234567` | Обновляет профиль клиента |

## Negative and edge cases
| ID | Input | Expected |
|---|---|---|
| E-01 | Несуществующая услуга | Не падает, предлагает ближайшие валидные услуги |
| E-02 | Опечатка в локации | Пытается сопоставить; если неуверенно, задает уточнение |
| E-03 | Off-topic запрос | Коротко отвечает и мягко возвращает к возможностям |
| E-04 | Токсичный запрос | Деэскалация, без конфликта |
